## Features
*   **Dynamic Content Management**: Projects, blogs, and technical stacks are managed through Nuxt Content, allowing for flexible and maintainable content updates.
*   **Responsive Design**: A seamless user experience across various devices and screen sizes.
*   **Appearance Customization**: Users can toggle between light and dark themes for personalized viewing.
*   **Project Filtering**: Filter projects by technology stack, showcasing relevant work efficiently.
*   **Detailed Project & Blog Pages**: Dedicated pages for each project and blog post, complete with structured content and navigation.
*   **Interactive Resume Viewer**: A convenient way to view the author's resume directly within the application.
*   **SEO Optimized**: Configured for search engine visibility, ensuring discoverability of content.
*   **Modern UI Components**: Utilizes Shadcn-vue and Tailwind CSS for a clean, consistent, and visually appealing interface.

## Getting Started

### Installation
To set up this portfolio locally, follow these steps:

*   **Clone the Repository**:
    ```bash
    git clone https://github.com/iamArvy/portfolio.git
    ```
*   **Navigate to the Project Directory**:
    ```bash
    cd portfolio
    ```
*   **Install Dependencies**: Use your preferred package manager (pnpm, npm, or yarn).
    ```bash
    pnpm install
    # or npm install
    # or yarn install
    ```
*   **Start the Development Server**:
    ```bash
    pnpm dev
    # or npm run dev
    # or yarn dev
    ```
    The application will typically be accessible at `http://localhost:3000`.

### Environment Variables
This project does not require specific environment variables for local development beyond standard Nuxt configurations.

## Usage
Once the development server is running, you can interact with the portfolio:

*   **Navigation**: Use the sidebar to navigate between sections like Profile, Projects, and Blog.
*   **Explore Projects**: Visit the `/projects` page to view a list of showcased projects. Click on any project to see its detailed description, technologies used, and links to live demos or repositories.
*   **Read Blogs**: The `/blog` section offers technical articles and insights. Each post provides in-depth information on various topics.
*   **Filter Content**: On the Projects page, use the filter option to narrow down projects by specific technology stacks.
*   **View Resume**: Click on the "View Resume" button (if available) to open an interactive PDF viewer of the author's resume.
